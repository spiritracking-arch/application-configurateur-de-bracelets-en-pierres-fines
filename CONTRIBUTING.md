# Contribuer à ce dépôt

Merci de l'intérêt porté à **GemStudio 360™** ! Ce dépôt étant une **vitrine technique et documentaire**, voici ce qui est bienvenu — et ce qui ne l'est pas.

## ✅ Contributions bienvenues

- Corrections et améliorations de la documentation (`README.md`, `docs/`)
- Ajout d'exemples pédagogiques supplémentaires (toujours simplifiés/illustratifs, jamais le code de production)
- Traductions du README et de la documentation (EN, ES, etc.)
- Suggestions d'architecture ou de bonnes pratiques (PWA, SEO, accessibilité, performance)
- Signalement de bugs constatés sur l'application en ligne (via une issue, avec capture d'écran et navigateur utilisé)
- Idées de fonctionnalités (voir [`docs/ROADMAP.md`](docs/ROADMAP.md))

## 🚫 Contributions non acceptées

- Tout code visant à reproduire, contourner ou imiter l'algorithme de calcul numérologique, le moteur de rendu génératif des pierres ou le système de score lithothérapique de production (propriété intellectuelle exclusive de Loch Ness®)
- Tout scraping ou extraction automatisée du catalogue complet de pierres ou des données de l'application en ligne
- Tout contenu publicitaire ou non lié au projet

## Processus

1. Ouvrez une issue pour discuter du changement avant une pull request conséquente.
2. Forkez le dépôt, créez une branche descriptive (`docs/ajout-section-faq`, `examples/ajout-demo-couleurs`...).
3. Soumettez votre pull request en expliquant le contexte et la motivation.

## Code de conduite

Soyez respectueux et constructif. Toute contribution doit rester dans l'esprit d'un dépôt vitrine documentaire public.
