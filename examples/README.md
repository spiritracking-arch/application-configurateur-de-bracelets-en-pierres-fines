# Exemples pédagogiques

⚠️ **Important** : les fichiers de ce dossier sont des **exemples simplifiés et illustratifs**, créés pour ce dépôt vitrine. Ils montrent le *principe* de fonctionnement de certains modules de GemStudio 360™, mais :

- ne contiennent pas la table de correspondance numérologique réelle ;
- ne contiennent pas le moteur de rendu génératif complet (presets par minéral, gestion des inclusions, du veinage, etc.) ;
- ne contiennent pas le catalogue réel des 70+ pierres ni les algorithmes de score lithothérapique.

| Fichier | Concept illustré |
|---|---|
| [`numerology-concept.example.js`](numerology-concept.example.js) | Principe : texte/date → valeur numérique → pierre symbolique |
| [`svg-bead-render.example.js`](svg-bead-render.example.js) | Principe : génération SVG d'une perle via gradient + seed aléatoire |
| [`stone-data-model.example.json`](stone-data-model.example.json) | Structure illustrative d'une fiche pierre dans le catalogue |
